## How to use the calendar

| Setting | Recommendation |
|---------|----------------|
| **Timezone** | Post in **Europe/Amsterdam (CET/CEST)** unless your audience is mostly still abroad—in that case, test **early morning NL** and **evening US**. |
| **Days** | **Tuesday, Wednesday, Thursday** usually outperform weekends for “admin” content; **Sunday evening** can work for “Monday I’m calling the gemeente” anxiety. This plan uses **weekdays Mon–Fri** over **4 weeks** so you can batch-write once. |
| **Times** | **08:15**, **12:45**, or **19:45** local—pick **one** slot per week and stay consistent for a month. |
| **Link** | Always use the **full URL** in the post (Facebook may strip tracking; that’s OK). UTMs help in GA4 when preserved. |
| **First comment** | If a group or Page strategy needs it, duplicate the link in the **first comment** after posting. |

## Groups: how to use this calendar (not simultaneous)

**Short answer: no.** Do **not** post the same starter post into **all** your groups at the **same** time (or the same day in a burst). Facebook and moderators treat that as **spam**, overlapping members see **duplicate** content, and you burn trust in communities where **comment-first** norms matter. Align with the caps in [facebook-outreach-and-groups.md](./facebook-outreach-and-groups.md) (help before links, **2–4× link posts per month per group** when rules allow—not every group allows any).

**Use three channels differently:**

| Channel | What the calendar means | Cadence |
|---------|-------------------------|---------|
| **Facebook Page** | **Follow the calendar day-for-day** (or the slower Mon+Thu option). This is your single **canonical** feed for the 20 posts. | 1 post per scheduled day |
| **Personal profile** | Optional: **share** from the Page, or repost with a personal line (“I wrote this after our gemeente day…”). Same dates as Page, or **offset by a few hours** so it doesn’t look robotic. | Same 20 pieces of content, not 20 *extra* unique essays unless you want that |
| **Groups** | The calendar is a **topic queue**, not “blast everywhere today.” | See table below |

**Group rules of thumb:**

1. **Participation first** in each group (answer questions **without** links until you’ve earned presence), unless that group explicitly welcomes resource drops.
2. **Hard cap (conservative):** At most **2 posts that include your link** per day **across all groups combined**—and many days should be **zero** (comments only).
3. **Per group:** Treat **2–4 link posts per month** as a ceiling where rules allow; **housing-heavy** groups often need **text-only** help or **mod approval** first.
4. **Spacing:** If the same post goes to a **second** group (audiences barely overlap), wait **at least 3–7 days** and tweak the opening line so it’s not byte-identical everywhere.
5. **Prefer replies:** Often the best “use” of Post #3 is to **answer** someone’s insurance thread with two sentences + **one** link—not a new wall post in six groups.

**UTMs in groups:** When you do drop a link in a specific group, swap `utm_medium=social` for `utm_medium=group` and add `utm_campaign=` with a short group slug (see the outreach doc). Keep the same `starter-XX-*` suffix if you want to tie back to which piece of content it was.

### Same weeks — how groups fit in (add-on, not parallel blast)

Use this **alongside** the table above. **Page:** publish Post # on that day. **Groups:** optional **extra** touchpoints, capped.

| Week | Mon | Tue | Wed | Thu | Fri |
|------|-----|-----|-----|-----|-----|
| **1** | Page **1** + **≤1** group link (pick *one* core Q&A group, only if rules allow) | Page **2** + **0–1** group link *or* comment-only day | Page **3** + comment help in 1–2 threads (link only if someone asked) | Page **4** + **≤1** group link (different group than Mon) | Page **5** + no new group wall posts (or 1 reply with link if highly relevant) |
| **2** | Page **6** + **≤1** group | Page **7** + **≤1** group (rotate roster) | Page **8** + comments | Page **9** + **≤1** group | Page **10** + comments |
| **3** | Page **11** + **≤1** group | Page **12** + **≤1** group | Page **13** + comments | Page **14** + **≤1** group | Page **15** + **≤1** group |
| **4** | Page **16** + **≤1** group | Page **17** + comments | Page **18** + **≤1** group | Page **19** + **≤1** group | Page **20** + comments |

**Reading the group cells:** “**≤1** group link” = **one** new post (or one approved promo thread) **in one** group that day, **not** the same paste in five groups. If you only have **3** groups you care about, most cells become **comment-only** or **share Page post into group** only where rules allow.

**Example roster (6 general expat groups):** Label them A–F. Week 1 Mon → group **A** gets Post 1; Week 1 Thu → **B** gets Post 4; Week 2 Mon → **C** gets Post 6; Week 2 Tue → **D** gets Post 7; … so each group receives **roughly 3–4** link posts over the month—inside a typical **2–4/month** norm if you skip some slots or use **comments** instead half the time.

**If you have no Page yet:** Still **do not** post all 20 into groups on a rigid daily grid without caps—use the **group** row logic only, **one group per slot**, and stretch to **8–10 weeks** so each group stays ≤2–4 links/month.
